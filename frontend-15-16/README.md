# Практические занятия №15-18: PWA "Flow" — Туду-лист с напоминаниями

## Описание
Приложение для управления задачами с расширенным PWA-функционалом, реализующее требования практических занятий №15-18:

### Практика №15 — HTTPS + App Shell
- **App Shell архитектура** — мгновенная загрузка каркаса приложения
- **Service Worker** — кэширование статики (Cache First) и динамических страниц (Network First)
- **Локальный HTTPS** — самоподписанный сертификат через `mkcert` для работы Service Worker
- **Два кэша**: `flow-shell-v1` (статика) и `flow-content-v1` (динамический контент)
- **Офлайн-режим** — приложение работает без интернета

### Практика №16 — WebSocket + Push
- **WebSocket (Socket.IO)** — двусторонняя связь в реальном времени между клиентами
- **Push-уведомления** — системные уведомления даже при закрытом приложении (Web-Push, VAPID)
- **Toast-уведомления** — всплывающие уведомления о новых задачах от других клиентов
- **Динамическая генерация VAPID ключей** — автоматическое создание и сохранение ключей
- **Эндпоинты подписки/отписки**: `/subscribe`, `/unsubscribe`
- **Кнопки управления уведомлениями** — включение/отключение push прямо из интерфейса

### Практика №17 — Детализация Push (Напоминания)
- **Форма напоминаний** — текст + дата/время (`datetime-local`)
- **Планирование на сервере** — `setTimeout` с хранением таймеров в `Map`
- **Push с кнопкой "Отложить"** — Service Worker обрабатывает действие `snooze`
- **Эндпоинт `/snooze`** — перенос напоминания на 5 минут
- **Бейдж напоминания** — визуальное отображение запланированного напоминания в задаче
- **Валидация времени** — напоминание только на будущее время

### Практика №18 — Подготовка к контрольной работе №3
- Тестирование работоспособности всех компонентов
- Данный README с описанием проделанной работы
- Открытый репозиторий для сдачи

## Установка и запуск

### 1. Установка зависимостей
```bash
npm install
```

### 2. Запуск сервера
```bash
node server.js
```
Сервер запустится на `http://localhost:3001`

### 3. HTTPS (опционально)
Для работы Service Worker в некоторых браузерах требуется HTTPS. Сертификаты уже сгенерированы через `mkcert`:
```bash
# Если mkcert установлен и доверен в системе:
mkcert -install
mkcert localhost 127.0.0.1 ::1
```

## Проверка работы

### WebSocket (Практика №16)
1. Откройте две вкладки браузера: `http://localhost:3001`
2. Добавьте задачу в первой вкладке
3. Во второй вкладке появится toast-уведомление о новой задаче

### Push-уведомления (Практика №16)
1. Нажмите "Включить уведомления" и разрешите их в браузере
2. Добавьте задачу в одной вкладке
3. В другой вкладке (или при закрытом приложении) придёт системное push-уведомление

### Напоминания (Практика №17)
1. Нажмите "Добавить с напоминанием"
2. Введите текст и выберите дату/время в будущем
3. В назначенное время придёт push-уведомление с кнопкой "Отложить на 5 минут"
4. Нажмите "Отложить" — напоминание перенесётся на 5 минут

### Service Worker (Практика №15)
1. Откройте DevTools → Application → Service Workers
2. Проверьте, что SW активен
3. В Cache Storage должно быть два кэша: `flow-shell-v1` и `flow-content-v1`

### Офлайн режим (Практика №15)
1. Отключите сеть в DevTools (Offline)
2. Обновите страницу — приложение загрузится из кэша
3. Задачи будут доступны, можно добавлять новые

## Технологии
- **Express.js** — сервер и раздача статики
- **Socket.IO** — WebSocket для реального времени
- **Web-Push** — push-уведомления с VAPID
- **Service Worker** — кэширование, офлайн, push-обработка
- **localStorage** — хранение задач и заметок на клиенте
- **mkcert** — локальные HTTPS сертификаты
- **CSS custom properties** — тёмная тема, минималистичный дизайн

## Архитектура

### App Shell
`index.html` содержит минимальный каркас (шапка, навигация, контейнер). Контент подгружается динамически через `fetch` в `app.js`.

### Стратегии кэширования
- **Cache First** — статика (HTML, CSS, JS, иконки, манифест)
- **Network First** — динамические страницы (`/content/*`)

### Модель данных задачи
```javascript
{
    id: 1712345678901,       // timestamp
    text: "Текст задачи",
    completed: false,         // статус выполнения
    note: "Заметка к задаче", // пользовательская заметка
    reminder: 1712349278901   // timestamp напоминания (null если нет)
}
```

### Поток напоминаний
1. Клиент → `socket.emit('newReminder', {id, text, reminderTime})`
2. Сервер → `setTimeout` с задержкой `reminderTime - Date.now()`
3. Сервер → `webpush.sendNotification()` + `io.emit('taskAdded')`
4. Клиент получает push с кнопкой "Отложить на 5 минут"
5. Клик → `fetch('/snooze?reminderId=...')` → сервер перепланирует на 5 мин
