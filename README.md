# Контрольная работа №2
## Фронтенд и бэкенд разработка (Часть 2)

Данный проект представляет собой полноценное веб-приложение для управления товарами с системой аутентификации и разграничением прав доступа (RBAC).

### Основные возможности:
1.  **Аутентификация:** Регистрация и вход пользователей с использованием JWT (Access + Refresh токены).
2.  **Ролевая модель (RBAC):**
    *   `guest` (неавторизованный): Просмотр списка товаров.
    *   `user`: Просмотр списка и деталей товаров.
    *   `seller`: Добавление и редактирование товаров.
    *   `admin`: Полный контроль над товарами и просмотр списка пользователей.
3.  **Безопасность:** Пароли хешируются с помощью `bcrypt`. Токены автоматически обновляются через интерцепторы `axios`.
4.  **Документация:** Swagger API документация доступна на бэкенде.

### Структура проекта:
*   `/server`: Node.js + Express сервер.
*   `/client`: React.js приложение.

### Запуск проекта:

#### Сервер:
1. Перейдите в папку `server`: `cd server`
2. Установите зависимости: `npm install`
3. Запустите сервер: `node server.js`
   * Сервер будет доступен по адресу `http://localhost:3000`
   * Swagger документация: `http://localhost:3000/api-docs`

#### Клиент:
1. Перейдите в папку `client`: `cd client`
2. Установите зависимости: `npm install`
3. Запустите клиент: `npm start`
   * Приложение откроется по адресу `http://localhost:3000` (в режиме разработки CRA может использовать другой порт, если 3000 занят).
